## 프로그램 철학

### 핵심 원칙
**"코드보다 생각, 구현보다 기획"**

바이브 코딩 시대에는 AI가 코드를 작성합니다. 
학생들에게 정말 필요한 것은:
- 문제를 정의하는 능력
- 사용자를 이해하는 공감력
- 솔루션을 설계하는 기획력
- 아이디어를 전달하는 소통력

### 수업 목표
1. **문제 발견 능력**: 일상에서 해결할 가치가 있는 문제를 찾는다
2. **사용자 중심 사고**: 사용자 관점에서 생각하고 공감한다
3. **기획 및 설계 능력**: 솔루션을 구체적으로 설계한다
4. **실행력**: 기획을 실제 프로토타입으로 만든다
5. **소통 및 발표력**: 아이디어를 효과적으로 전달한다

## 9학년: 자기표현과 창의적 문제 발견

### 주제
**"나를 표현하는 디지털 포트폴리오 기획하기"**

### 학습 목표
- 자신의 강점과 관심사를 탐색하고 표현할 수 있다
- 정보를 구조화하고 시각적으로 설계할 수 있다
- 사용자 경험(UX) 관점에서 생각할 수 있다
- 기획서를 작성하고 발표할 수 있다

### 1차시: 자기 탐색 및 문제 발견

#### 수업 목표

- "10년 후 여러분은 어떤 모습일까?"

**우수 사례 공유**
- 인상적인 포트폴리오 웹사이트 3-5개 시연
- 각 사례의 특징과 장점 분석

│    ④ _______________________        │
│    ⑤ _______________________        │

│                                     │

│    ___________________________      │

**활동 2: 페르소나 만들기 (15분)**

- 나를 표현하는 이미지 5장 수집

### 2차시: 정보 구조 설계 (IA)

#### 수업 목표
웹사이트의 정보를 논리적으로 구조화하기

#### 도입 (10분)
**정보 구조(Information Architecture)란?**
- 정보를 조직하고 배치하는 방법
- 사용자가 원하는 정보를 쉽게 찾도록 돕기

**좋은 구조 vs 나쁜 구조**
- 사례 비교 분석
- 사용자 입장에서 생각하기

#### 전개 (30분)

**활동 1: 콘텐츠 인벤토리 (10분)**

```
┌─────────────────────────────────────┐
│      콘텐츠 인벤토리                 │
├─────────────────────────────────────┤
│ 포트폴리오에 포함할 내용:           │
│                                     │
│ □ 프로필 사진                       │
│ □ 이름 및 소개                      │
│ □ 학력/경력                         │
│ □ 관심사/취미                       │
│ □ 프로젝트/작품                     │
│ □ 스킬/능력                         │
│ □ 수상 경력                         │
│ □ 연락처                            │
│ □ SNS 링크                          │
│ □ 기타: _______________            │
└─────────────────────────────────────┘
```

**활동 2: 사이트맵 작성 (20분)**

```
사이트맵 예시:

홈 (Home)
├── 소개 (About)
│   ├── 프로필
│   ├── 이야기
│   └── 가치관
├── 관심사 (Interests)
│   ├── 취미
│   ├── 활동
│   └── 학습
├── 프로젝트 (Projects)
│   ├── 프로젝트 1
│   ├── 프로젝트 2
│   └── 프로젝트 3
└── 연락 (Contact)
    ├── 이메일
    └── SNS
```

**작성 가이드**
1. 메인 페이지 정하기 (보통 4-6개)
2. 각 페이지의 하위 내용 정리
3. 페이지 간 연결 관계 표시
4. 사용자 동선 고려

#### 정리 (10분)
**동료 리뷰**
- 옆 사람과 사이트맵 교환
- 피드백 질문:
  * 구조가 논리적인가?
  * 찾고 싶은 정보를 쉽게 찾을 수 있는가?
  * 빠진 내용은 없는가?

#### 과제
- 사이트맵 수정 및 보완
- 각 페이지에 들어갈 내용 구체화

### 3차시: 와이어프레임 스케치

#### 수업 목표
페이지 레이아웃을 시각적으로 설계하기

#### 도입 (10분)
**와이어프레임이란?**
- 웹사이트의 뼈대 그림
- 내용 배치와 구조 표현
- 디자인 전 단계의 설계도

**와이어프레임 vs 디자인**
- 와이어프레임: 구조와 배치 (흑백, 간단)
- 디자인: 색상, 폰트, 이미지 (컬러, 상세)

#### 전개 (30분)

**활동 1: 레이아웃 패턴 학습 (10분)**

```
일반적인 웹 레이아웃 패턴:

1. F-패턴
   ┌─────────────────┐
   │ ███████████     │  ← 상단 네비게이션
   │ ███             │  ← 왼쪽 콘텐츠
   │ ████████        │  ← 중간 콘텐츠
   │ ███             │
   └─────────────────┘

2. Z-패턴
   ┌─────────────────┐
   │ ████████████    │  ← 상단
   │         ↘       │
   │       ↙         │
   │ ████████████    │  ← 하단
   └─────────────────┘

3. 그리드 레이아웃
   ┌─────────────────┐
   │ ┌───┐ ┌───┐ ┌───┐│
   │ │   │ │   │ │   ││
   │ └───┘ └───┘ └───┘│
   │ ┌───┐ ┌───┐ ┌───┐│
   │ │   │ │   │ │   ││
   │ └───┘ └───┘ └───┘│
   └─────────────────┘
```

**활동 2: 와이어프레임 스케치 (20분)**

**준비물**: A4 용지, 펜/연필

**스케치 가이드**
1. 홈페이지부터 시작
2. 주요 요소 배치
   - 헤더 (로고, 메뉴)
   - 메인 콘텐츠
   - 사이드바 (선택)
   - 푸터
3. 각 페이지별로 스케치
4. 화살표로 연결 관계 표시

**와이어프레임 체크리스트**
```
□ 모든 주요 페이지 스케치 완료
□ 네비게이션 메뉴 위치 표시
□ 콘텐츠 영역 명확히 구분
□ 버튼/링크 위치 표시
□ 이미지 위치 표시
□ 텍스트 영역 표시
```

#### 정리 (10분)
**갤러리 워크**
- 모든 학생의 스케치를 벽에 붙이기
- 자유롭게 돌아다니며 관찰
- 좋은 아이디어 포스트잇에 메모

#### 과제
- 와이어프레임 디지털화 (Figma, PowerPoint 등)
- 모바일 버전도 고려하기

### 4차시: 사용자 경험(UX) 설계

#### 수업 목표
사용자 관점에서 경험을 설계하기

#### 도입 (10분)
**UX(User Experience)란?**
- 사용자가 제품/서비스를 사용하며 느끼는 전체 경험
- 사용하기 쉽고, 즐겁고, 효과적인 경험 만들기

**좋은 UX vs 나쁜 UX**
- 사례 비교
- 사용자 입장에서 생각하기

#### 전개 (30분)

**활동 1: 사용자 여정 지도 (User Journey Map) (20분)**

```
┌─────────────────────────────────────────────────────────┐
│              사용자 여정 지도 템플릿                     │
├─────────────────────────────────────────────────────────┤
│                                                          │
│ 시나리오: 처음 방문한 사람이 나에 대해 알아가는 과정    │
│                                                          │
│ 단계 1: 첫 방문                                         │
│ ├─ 행동: 홈페이지 접속                                  │
│ ├─ 생각: "이 사람은 누구지?"                            │
│ ├─ 감정: 😐 중립                                        │
│ └─ 터치포인트: 메인 비주얼, 소개 문구                   │
│                                                          │
│ 단계 2: 탐색                                            │
│ ├─ 행동: 메뉴 클릭, 페이지 이동                         │
│ ├─ 생각: "어떤 것을 잘하는 사람일까?"                   │
│ ├─ 감정: 🤔 궁금함                                      │
│ └─ 터치포인트: 네비게이션, 프로젝트 카드                │
│                                                          │
│ 단계 3: 깊이 이해                                       │
│ ├─ 행동: 프로젝트 상세 보기                             │
│ ├─ 생각: "흥미로운 사람이네!"                           │
│ ├─ 감정: 😊 긍정적                                      │
│ └─ 터치포인트: 프로젝트 설명, 이미지                    │
│                                                          │
│ 단계 4: 행동                                            │
│ ├─ 행동: 연락하기                                       │
│ ├─ 생각: "연락해봐야겠다"                               │
│ ├─ 감정: 😃 만족                                        │
│ └─ 터치포인트: 연락 버튼, 이메일                        │
└─────────────────────────────────────────────────────────┘
```

**작성 가이드**
1. 주요 사용자 시나리오 정의
2. 각 단계별 사용자 행동 예상
3. 각 단계별 사용자 생각/감정 추측
4. 개선 포인트 찾기

**활동 2: 인터랙션 설계 (10분)**

```
인터랙션 설계 체크리스트:

□ 버튼 호버 시 반응 (색상 변화, 크기 변화 등)
□ 페이지 전환 애니메이션 (부드러운 이동)
□ 스크롤 효과 (페이드인, 슬라이드 등)
□ 로딩 상태 표시
□ 에러 메시지 (친절한 안내)
□ 성공 피드백 (완료 알림)
□ 모바일 터치 제스처
```

#### 정리 (10분)
**UX 개선 아이디어 공유**
- 각자 가장 중요한 UX 개선 포인트 1가지 발표
- 클래스 전체 투표로 베스트 아이디어 선정

#### 과제
- 사용자 여정 지도 완성
- 인터랙션 상세 설계

### 5차시: 비주얼 디자인 기획

#### 수업 목표
시각적 아이덴티티를 기획하기

#### 도입 (10분)
**디자인의 중요성**
- 첫인상은 3초 안에 결정
- 시각적 요소가 전달하는 메시지
- 일관성 있는 디자인의 힘

#### 전개 (30분)

**활동 1: 무드보드 만들기 (15분)**

```
┌─────────────────────────────────────┐
│          무드보드 템플릿             │
├─────────────────────────────────────┤
│                                     │
│ 1. 컨셉 키워드 (3개)                │
│    예: 모던, 미니멀, 따뜻한          │
│    ___________________________      │
│                                     │
│ 2. 색상 팔레트 (3-5개)              │
│    ┌───┐ ┌───┐ ┌───┐              │
│    │   │ │   │ │   │              │
│    └───┘ └───┘ └───┘              │
│    #_____ #_____ #_____            │
│                                     │
│ 3. 폰트 스타일                      │
│    제목: _______________            │
│    본문: _______________            │
│                                     │
│ 4. 레퍼런스 이미지                  │
│    (좋아하는 디자인 스타일 이미지)  │
│    [이미지 붙이기]                  │
│                                     │
│ 5. 아이콘 스타일                    │
│    □ 라인형  □ 채움형              │
│    □ 둥근형  □ 각진형              │
└─────────────────────────────────────┘
```

**색상 선택 가이드**
- 메인 컬러 1개: 브랜드를 대표하는 색
- 보조 컬러 2-3개: 메인 컬러와 조화
- 배경 컬러: 흰색, 회색 등 중립색
- 강조 컬러: 버튼, 링크 등에 사용

**활동 2: 스타일 가이드 작성 (15분)**

```
┌─────────────────────────────────────┐
│         스타일 가이드                │
├─────────────────────────────────────┤
│                                     │
│ 색상 규칙:                          │
│ - 메인 컬러: 제목, 버튼, 강조       │
│ - 보조 컬러: 아이콘, 구분선         │
│ - 배경 컬러: 페이지 배경            │
│                                     │
│ 타이포그래피:                       │
│ - H1 (페이지 제목): 32px, Bold     │
│ - H2 (섹션 제목): 24px, Bold       │
│ - 본문: 16px, Regular               │
│ - 캡션: 14px, Light                 │
│                                     │
│ 간격 규칙:                          │
│ - 섹션 간격: 80px                   │
│ - 요소 간격: 40px                   │
│ - 텍스트 줄간격: 1.6                │
│                                     │
│ 버튼 스타일:                        │
│ - 크기: 높이 48px                   │
│ - 모서리: 둥글게 8px                │
│ - 호버: 색상 10% 어둡게             │
└─────────────────────────────────────┘
```

#### 정리 (10분)
**무드보드 갤러리**
- 각자의 무드보드 공유

## 교사 가이드

### 수업 운영 원칙

#### 1. 학생 주도 학습
```
교사의 역할:
❌ 답을 알려주는 사람
✅ 질문으로 생각을 이끄는 코치

예시:
학생: "이 아이디어 어때요?"
교사: "누가 이 문제로 고통받나요?"
      "왜 이 솔루션이 필요한가요?"
      "다른 방법은 없을까요?"
```

#### 2. 실패 경험 장려
```
실패는 학습의 기회:
- "실패했다"가 아니라 "배웠다"
- "안 된다"가 아니라 "다른 방법을 찾자"
- 실패 사례 공유 시간 마련
```

#### 3. 협업 문화 조성
```
협업 촉진 방법:
- 팀 역할 명확히 정의
- 정기적 팀 회고 시간
- 갈등 해결 프로세스 마련
- 개인 평가 + 팀 평가 병행
```

4. **아이디어를 전달하는 입**
5. **실행으로 옮기는 손**
